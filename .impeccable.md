# Buglist Design System

## Design Context

### Users
**Who**: 测试工程师和开发人员，他们需要快速生成和执行 UI 自动化测试用例。

**Context**: 用户在工作中使用这个平台来提高测试效率，从产品文档生成测试用例，通过 Playwright 执行测试，并使用 AI 视觉能力进行结果判断。他们需要一个既专业又令人兴奋的工具。

**Job to be done**:
- 快速导入或生成测试用例
- 实时监控测试执行进度
- 清晰查看测试结果和报告
- 配置 AI 和集成服务

### Brand Personality
**Voice & Tone**: 创新、前卫、充满科技感，但不失专业性。像一个来自未来的测试工具。

**3-Word Personality**:
1. **创新 (Innovative)** - 突破传统测试工具的设计边界
2. **前卫 (Cutting-edge)** - 使用最新的视觉设计语言
3. **精致 (Refined)** - 每个细节都经过精心打磨

**Emotional Goals**:
- 让用户感到兴奋和惊艳（第一印象）
- 传达技术先进性和可靠性
- 提供流畅、愉悦的使用体验
- 激发用户的创造力和探索欲

### Aesthetic Direction

**Visual Tone**: 现代科技感，融合玻璃态设计（Glassmorphism）和流动渐变，创造出未来感和深度感。

**Color System**:
- **主色调**: 多色渐变系统
  - 紫-蓝渐变 (#8B5CF6 → #3B82F6) - 主要交互元素
  - 粉-橙渐变 (#EC4899 → #F59E0B) - 强调和警告
  - 青-绿渐变 (#06B6D4 → #10B981) - 成功状态
  - 深蓝-紫渐变 (#1E1B4B → #4C1D95) - 背景和容器
- **玻璃态**: 半透明背景 + 模糊效果 + 细微边框
- **发光效果**: 使用 box-shadow 和 filter: drop-shadow 创造霓虹感

**Key Visual Elements**:
- **玻璃态卡片**: backdrop-filter: blur(20px), 半透明背景, 细微边框高光
- **流动渐变**: 使用 CSS 动画让渐变背景流动
- **粒子效果**: 背景中的动态粒子系统
- **3D 变换**: 卡片悬停时的 3D 倾斜效果
- **发光边框**: 交互元素的霓虹发光效果
- **视差滚动**: 不同层级的元素以不同速度移动

**Animation Philosophy**:
- **极致炫酷**: 大量使用动画效果，但保持流畅（60fps）
- **微交互**: 每个交互都有精致的反馈动画
- **入场动画**: 元素以优雅的方式进入视野
- **悬停效果**: 丰富的 hover 状态，包括 3D 变换、发光、缩放
- **进度动画**: 使用粒子、波浪、脉冲等效果展示进度

**Typography**:
- **主字体**: Outfit - 现代、几何、易读
- **等宽字体**: JetBrains Mono - 代码、数字、技术信息
- **字重层次**: 300 (light) / 400 (regular) / 500 (medium) / 600 (semibold) / 700 (bold)

**References** (类似的视觉风格):
- Linear.app - 流畅的动画和渐变
- Stripe.com - 玻璃态和现代感
- Vercel.com - 简洁但充满细节
- Apple Vision Pro 界面 - 玻璃态和深度感
- Framer Motion 官网 - 极致的动画效果

**Anti-references** (避免的风格):
- 传统企业软件的沉闷界面
- 过度拟物化的设计
- 纯扁平化、缺乏层次感
- 过于花哨、影响可读性的设计

### Design Principles

#### 1. 视觉优先，但不牺牲功能
每个设计决策都应该首先考虑视觉冲击力，但必须确保功能性和可用性不受影响。炫酷的效果应该增强而不是干扰用户的工作流程。

**Why**: 用户希望工具既炫酷又实用，视觉设计应该服务于更好的用户体验。

**How to apply**:
- 动画应该有意义，不是为了动而动
- 玻璃态效果不应影响内容的可读性
- 渐变色彩应该有足够的对比度
- 所有交互元素都应该清晰可见

#### 2. 流动性和连贯性
界面应该像流体一样流畅，所有元素之间应该有连贯的视觉语言。渐变、动画、过渡都应该是平滑和自然的。

**Why**: 流畅的体验让用户感到愉悦，连贯的设计语言建立信任感。

**How to apply**:
- 使用 cubic-bezier 缓动函数创造自然的动画
- 渐变应该平滑过渡，避免突兀的颜色跳跃
- 页面切换和状态变化应该有过渡动画
- 保持一致的圆角、间距、阴影系统

#### 3. 深度和层次
通过玻璃态、阴影、模糊、3D 变换创造丰富的视觉层次，让界面有深度感和空间感。

**Why**: 深度感让界面更有吸引力，层次感帮助用户理解信息架构。

**How to apply**:
- 使用 backdrop-filter 创造玻璃态效果
- 多层阴影系统（近、中、远）
- 悬停时的 3D 倾斜效果（transform: perspective + rotateX/Y）
- 背景层、内容层、浮动层的明确区分

#### 4. 性能至上
所有动画和效果都必须保持 60fps，使用 GPU 加速，避免影响页面性能。

**Why**: 再炫酷的效果如果卡顿就会破坏体验，性能是用户体验的基础。

**How to apply**:
- 优先使用 transform 和 opacity 做动画
- 使用 will-change 提示浏览器优化
- 粒子效果使用 Canvas 或 WebGL
- 大型动画使用 requestAnimationFrame
- 提供降低动画的选项（prefers-reduced-motion）

#### 5. 渐进增强
从简洁的基础开始，逐步添加炫酷效果。确保在不支持高级特性的浏览器中仍然可用。

**Why**: 不是所有用户都有高性能设备，但所有用户都应该能使用核心功能。

**How to apply**:
- 使用 @supports 检测特性支持
- 玻璃态效果有降级方案（纯色背景）
- 复杂动画在低性能设备上简化
- 核心功能不依赖于视觉效果
